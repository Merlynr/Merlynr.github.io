Linux移植入门

发表于2022-11-17|更新于2022-11-17|graduate student time
|字数总计:453|阅读时长:1分钟|阅读量:

典型分区结构

bootloader、kernel、rootfs、data fs

Bootloader

基本功能

  • 初始化硬件：
  • 将操作系统内核从Flash:拷贝到SDRAM中；
  • 改写系统的内存映射，原先Flash起始地址映射为O地址，这时需要将RAM的起始地址映射为0;
  • 设置堆栈指针并将bss段清零，执行C语言程序和调用子函数时要用到：
  • 提供Linux内核的启动参数；
  • 启动Linux内核；
  • 改变pc值，使得CPU开始执行真正的操作系统内核。

Linux kernel

功能及本质

功能：进程管理、文件系统、内存管理、设备管理、网络协议


内存管理：裸板上是物理地址，基于Linux系统则是虚拟地址
进程管理：宏观并行、微观串行
文件系统：虚拟文件系统。emc

内核本质：
从C语言角度来看内核，C语言函数集合；
从硬件角度来看内核，管理者和维护者；
从应用程序的角度来看内核，服务提供者；

data fs 嵌入式文件系统

概述

  1. 是一种对存储设备上的数据进行组织和控制的机制。
  2. 是操作系统的一部分，负责管理和存储文件信息。

三者之间关系

Bootloader

  • 硬件上电后跳到一个固定位置执行相应的代码
  • 初始化相应的硬件设备
  • 加载操作系统内核代码到内存
  • 跳到内核代码起始位置执行

Kernel

  • 内核自解压(ulmage)
  • 初始化相应的硬件设备
  • 初始化静态编译进内核的驱动模块I
  • 挂载根文件系统
  • 直接执行第一个用户空间程序

第一个用户空间程序

  • 配置用户环境和执行服务进程
